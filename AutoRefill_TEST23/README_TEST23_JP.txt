[TEST23]
AutoRefill TEST23
========================

概要
----
TEST22 の左手補充成功版を土台に、右手ホットバー 1〜9 の消費起点補充を追加した版です。
左手補充のロジックは残しつつ、ホットバー各スロットで「前tickはアイテムあり / 今tickで空」を検知したとき、
インベントリ内の完全一致アイテムを探して、空になった元スロットへ補充を試みます。

対象
----
- 左手補充
- 右手ホットバー 1〜9 補充

今回の意図
----------
- TEST22 の左手成功を壊さずに進める
- 右手全スロット補充の最小実装を追加する
- 完全一致判定は維持する

補充条件
--------
- 直前 tick ではアイテムあり
- 現在 tick で空になった
- インベントリ内に完全一致アイテムがある

未対応寄り
----------
- 水バケツ / 溶岩バケツのような「使用後に別アイテムへ変化するもの」
- 右手補充の専用例外処理（将来候補）

反応
----
- 成功: 成功メッセージ + 成功音
- 候補なし: NO CANDIDATE + 失敗音
- 試行失敗: ATTEMPTED BUT FAILED + 失敗音
- いずれもログ出力あり

ビルド
------
PowerShellでこのフォルダを開いて、1行ずつ実行

    .\gradlew.bat clean
    .\gradlew.bat build

できた jar は build\libs に入ります。

今回の変更メモ
--------------
- TEST22 の左手補充ロジックは維持
- ホットバー 1〜9 の前回状態スナップショットを追加
- 空になったホットバー番号を検知して、そのスロットへ補充を試みる処理を追加
- 完全一致判定はそのまま維持
